Linux的定时任务服务crond（crontab）服务

发表于 2017-09-13 | 分类于 Linux

crond是什么？ crond是linux系统中定期执行命令或者指定程序任务的一种服务 crond默认每分钟会检查系统中是否有需要执行的定时任务，如果有就会根据事先定义好的规则执行任务 秒级定时任务：crond改进、 自写守护进程脚本实现 、quartz实现 linux定时任务分类： 系统自身的定期执行的任务（ll /etc/|grep cron） 用户执行的定时任务 crontab 命令介绍： -u 指定使用用户的定时任务，不指定即当前用户 -l 查看定时任务 -r 删除定时任务 -e 修改定时任务（实际编辑的文件/var/spool/cron/用户名） -i 删除前提示确认 常见文件 /etc/cron.deny 该文件中所列用户不允许使用crontab命令 /etc/cron.allow 该文件中所列用户允许使用crontab，优先于/etc/cron.deny /var/spool/cron/ 所有用户crontab配置文件默认存放此目录，文件名以用户名命名 /var/log/cron 日志文件 定时任务添加格式 用户定时任务一般分为6段（用空格分割，系统定时任务/etc/crontab，分为8段），前五段为时间设定，第6段为执行的脚本或者命令段 分 时 日 月 周 符号: 表示任意时间 例子：00 23 cmd 每月每周每天的23:00执行cmd任务 - 表示一个时间范围 例子：00 17-19 cmd 每月每周每天17 18 19点整执行cmd任务 , 表示分隔时段 例子：30 3,6,9 cmd 每月每周每天的3,6,9点30分执行cmd任务 /n 表示每隔n单位时间，n代表数字 例子：/10 * cmd 每隔十分钟执行一次cmd任务

linux用户管理

发表于 2017-09-14 | 分类于 Linux 
用户分类：超级用户 普通用户 虚拟用户
  • 用户UID范围：

0 超级用户 1-499 虚拟用户 500-65535 普通用户

  • 用户与组相关文件： /etc/passwd（用户配置文件）/etc/shadow（用户影子口令文件）/etc/group（用户组配置文件）/etc/gshadow（用户组影子文件）

/etc/passwd文件内容格式：账号名:账号密码:账号UID:组UID:用户说明:用户家目录:shell解释器 /etc/shadow文件内容格式：账户名:账号密码:最近修改密码时间:禁止修改密码天数:用户必须改密天数:警告改密天数:不活动时间:失效时间:标志（保留） /etc/gshadow文件内容格式：用户组名:用户组密码:用户组管理员:用户组成员 /etc/group文件内容格式：用户组名:组密码:组UID /etc/skel/ 存放新用户的配置文件，当添加新用户时该目录下所有问价将复制到新用户家目录 /etc/login.defs 配置用户、组、密码相关属性的文件 /etc/default/useradd 存放添加新用户的默认配置 GROUP=100 GID HOME=/home 家目录 INACTIVE=-1 是否张那个号过期-1不启用 EXPIRE= 账号终止日期 SHELL=/bin/bash 默认shell SKEL=/etc/skel 配置新用户家目录文件存放路径 CREATE_MAIL_SPOOL=yes 创建mail文件 使用useradd添加用户时调用的一个默认配置文件 也可以使用useradd -D 参数 修改 用户管理命令汇总 （常用） useradd 添加用户 userdel 删除用户 passwd 设置用户密码 （技巧：echo “密码”|passwd –stdin 用户名 不需要交互适合大批量改密） chage 修改用户密码有效期 usermod 修改用户 id 查看用户UID、GID、用户组 su 用户切换 sudo 通过另一个用户执行命令 w 显示当前登录所有用户 who 当前账户 lastlog 登录信息 last （不常用） visudo 配置sudo权限，也可直接vi编辑/etc/sudoers pwcov 同步用户从/etc/passwd到/etc/shadow pwunconv是pwcov逆向操作 chfn 更改用户信息工具 chsh 更改用户登录shell工具 finger 查看用户信息工具 sudoedit和sudo差不多 组管理命令汇总 （常用） groupadd 添加用户组 gpasswd 设置用户组密码 groups 显示用户所属组 newgrp 更改用户所属的有效组 （不常用） grpconv 通过/etc/group和/etc/gshadow的文件同步创建/etc/gshadow，如果不存在则创建 grpunconv 通过/etc/group和/etc/gshadow的文件同步创建/etc/group，然后删除/etc/gshadow groups 查看当前用户组 重要命令详解（号必须掌握） useradd -c 新账号的说明栏 -d 家目录 -e 账号终止日期，格式MM/DD/YY -f 账号过期几日停权，0立刻、-1关闭此功能，默认-1 -g 用户组 -G 设置此用户多个组，用,分割 -m 用户目录不存在则自动创建 -M 不建立家目录 -n 默认用户用户组名相同，此参数就不会生成相同的 -r 建立系统账号，默认无家目录，如需要则加-m -s 指定用户登录shell -u 指定uid useradd -D -b 家目录前一个目录 -e 账号停止日期 -f 账号过期几日后停权 -g 新账号其实用户组id -s 登录shell groupadd -g 指定GID -r 建立系统用户组 -f 强制建立一个已经存在的组 passwd -l 锁定用户 -u 解除锁定 -n 多少天不能修改密码 -x 多少天后必须改密 -w 过期前几天通知用户 -i 过期几天禁止登陆 -S 查看用户密码状态，仅root有权限 -d 清除密码 –stdin 从stdin读入密码 chage -d 将最近一次密码设置时间设为“最近日期” -E 设置账户过期时间（MM/DD/YY） -l 查看账户信息 -I 过期几天数后停权 -m 两次改密最小天数 -M 两次改密最大天数 -W 过期警告天数 -R chroot 到的目录 userdel -r 删除主目录和邮件池 groupdel usermod -c 注释 -d 用户的新主目录 -e 设定帐户过期的日期MM/DD/YY -f 过期几天数后，设定密码为失效状态 -g 更新新用户心得起始登录组 -G 定义用户加入多个组 -a 将用户追加至上边 -G 中提到的附加组中，并不从其它组中删除此用户 -l 新的登录名称 -L 锁定用户帐号 -m 将家目录内容移至新位置 (仅于 -d 一起使用) -o 允许使用重复的(非唯一的) UID -p 将加密过的密码设为新密码 -s 该用户帐号的新登录 shell -u *用户帐号的新 UID -U 解锁用户帐号
